I built a Cursor skill called session-journal that automatically chronicles each day's development story into a living markdown document and a structured JSON file. The first version logged every prompt individually like telemetry, but that felt soulless. Then came the real prize — turning that journal into a feature. I created the DevLog view, a lore-themed timeline woven into the Voyage section, gated behind developer mode. The IPC plumbing was straightforward — a tiny handler in the Electron main process reads the JSON, the preload exposes it as `window.devlog`, and the React component fetches on mount. Clean, minimal, dev-mode only.

Later in the day, I did a full UX rethink of the Transmissions page. The old layout had a permanent side-by-side split — the Decode Bay (BrowserView reader) sat there empty when idle, wasting half the screen. That bugged me. So I rebuilt the whole thing around a simple principle: **idle = full-width stream, decode = resizable split.** When no article is selected, the signal stream fills the entire viewport — every pixel is useful. The moment you select a transmission, the Decode Bay slides in from the right and a resizable drag divider appears between the stream and the reader. You can drag it to allocate space however you want, and Focus mode still goes full-screen reader. I also pulled the Scheduled Broadcasts section out of the permanent sidebar and made it a collapsible horizontal strip at the top of the stream, so it doesn't eat vertical space unless you want it. The loading skeleton now matches the full-width idle layout too. Zero wasted real estate.

Then the big one — **live event intelligence**. The old "Scheduled broadcasts" was just a static list of event names with naive word matching. Useless. I built a full background scraper in the Electron main process that fetches each event's official website using `net.fetch`, extracts dates from the page text using multi-pattern regex (US/EU date formats, ranges), and scrapes YouTube and Twitch links from `<a>` and `<iframe>` tags. Results are cached to a JSON file with a 24-hour TTL so the app doesn't hammer event sites on every launch. The renderer calls the scraper via IPC, gets back resolved events with start/end dates, and computes live status (upcoming / live / past / unknown) every 60 seconds. The UI now shows **live countdown badges** (e.g. "14d 3h", "2h 30m"), a pulsing **LIVE** indicator when an event is running, YouTube and Twitch quick-open buttons (which open in the Decode Bay, not external), and events are auto-sorted with live first, upcoming next, unknown, then past. Added official URLs for 20 of the 24 events. Zero new APIs — just smart scraping with existing Electron infrastructure.

All 43 tests cover date extraction (all formats), link extraction (YouTube/Twitch/dedup/decoding), URL validation, status computation, status refresh referential stability, and event sorting logic.

## Tuesday, February 24, 2026

*progress, fix, challenge, milestone*

Big day of polish and deep stability work. Accent colors unified, ML inference fixed, guided tour realigned, Data Flow layout rebuilt, Oracle error state redesigned as an Ark terminal card, and galaxy neighbor connections capped at 0.9 cosine distance for tighter relationships.

The headline: the Galaxy Map finally works at scale. With 111K+ embeddings, the Embedding Space was crashing with OOM errors — the renderer was allocating a 456 MB Float32Array just to load raw 1024D vectors, on top of ~1.75 GB baseline memory. Multiple rounds of worker-side fixes (in-place centering, landmark subsampling, mutable refs) helped the worker but couldn't fix the main-thread allocation. The final solution was architectural: stream-project each 1024D vector to 100D during the IDB cursor read itself — L2 normalize, multiply by a Gaussian random projection matrix, store into a pre-allocated 42.5 MB buffer. The full 1024D data never exists in memory. Then run PCA (WebGPU-accelerated when available, CPU fallback) directly on the 100D buffer — fast, deterministic, zero OOM risk. The galaxy now builds 111K nodes in seconds with the renderer sitting at ~1.05 GB. Random projection preserves pairwise distances (Johnson-Lindenstrauss), so the 3D layout quality is maintained.

Also shipped: Ollama as an optional NSIS installer step (detects existing install, clear value messaging, GPU recommendation, graceful failure paths), recommendation pipeline 100x speedup (memoized genre lookup, pre-computed semantic scores, dense Float32Array cosine, smart pre-filter 25K to 3K), view switching lag eliminated (4 heavy views lazy-imported, async filter/sort path), and 11 resource pressure fixes (crash circuit breaker, event-driven IPC, lazy ML model, HDR texture cleanup).
